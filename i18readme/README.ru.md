# i18readme

Переводите ваш `README.md` на любой язык одной командой. На основе Claude или GPT-4o.

Языковая панель выше добавляется автоматически в каждый файл. Когда ваш README изменится, пересчитываются только устаревшие переводы.

---

## Установка

```bash
npm install -g i18readme
```

Требуется Node.js 18+.

---

## Быстрый старт

```bash
# 1. Перейдите в папку проекта
cd my-project

# 2. Инициализация — спросит языки, папку, AI-провайдера, модель и API ключ
i18r init

# 3. Переводите
i18r sync
```

Готово. Ваши переводы появятся в `i18readme/README.ru.md`, `i18readme/README.de.md` и так далее.

---

## Команды

```
i18r init                          Настроить проект (интерактивно)
i18r sync                          Перевести недостающие и устаревшие файлы
i18r sync --force                  Пересчитать все переводы
i18r sync --force ru,de            Пересчитать только определённые языки
i18r sync --provider <p>           Переопределить провайдера для этого запуска
i18r sync --model <m>              Переопределить модель для этого запуска
i18r status                        Показать, какие файлы актуальны, а какие нет
i18r config                        Изменить провайдера, модель и API ключ
i18r config show                   Показать сохранённые настройки
i18r config set --provider <p> --key <k>   Сохранить API ключ без интерактива
```

---

## Как это работает

**`i18r init`** сканирует ваш проект в поиске файлов README, спрашивает, на какие языки переводить, какого провайдера и модель AI использовать, и сохраняет ваш API ключ. Создаёт `.i18readme.json`:

```json
{
    "langs": ["ru", "de", "zh"],
    "dir": "i18readme",
    "readmes": ["README.md"]
}
```

**`i18r sync`** сравнивает хеш содержимого вашего `README.md` с хешем, сохранённым в каждом переводе. Если содержимое не изменилось, перевод пропускается — даже при переключении веток в git или копировании файлов, которые обнулили бы временные метки.

**Каждый переведённый файл** получает языковую панель в верхней части со ссылками на все остальные языки.

## AI-провайдеры

| Провайдер          | Модель по умолчанию        | Как получить ключ                                      |
| ------------------ | -------------------------- | ------------------------------------------------------ |
| Claude _(по умолч.)_ | claude-haiku-4-5 (быстро)  | [console.anthropic.com](https://console.anthropic.com) |
| OpenAI             | gpt-4o-mini (быстро)       | [platform.openai.com](https://platform.openai.com)     |

Вы можете выбрать более мощную модель (Sonnet, Opus, GPT-4o) при запуске `i18r init` или `i18r config`, или переопределить её для одного запуска с помощью `--model`.

Ключ хранится в `~/.i18readmerc` на вашей машине — никогда не в самом проекте.
